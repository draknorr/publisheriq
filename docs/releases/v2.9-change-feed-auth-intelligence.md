# v2.9 - Change Feed, Auth Hardening, and Change Intelligence

**Release Date:** March 15, 2026

Version 2.9 brings three connected improvements online:

- a new `/changes` Change Feed for tracking grouped storefront, PICS, media, and news activity
- hardened auth/session behavior for OTP-first login and redirect handling
- broader operational maturity in the change-intelligence runtime

---

## Highlights

### Change Feed

- New signed-in route at `/changes`
- Two tabs:
  - **Feed** for grouped change bursts
  - **News** for recent Steam news posts
- Presets for `High Signal`, `Upcoming Radar`, and `All Changes`
- Filters for time range, app type, source, and search
- Detail drawer with individual change events, related news, and impact windows
- Health badge showing `healthy`, `catching_up`, or `delayed`

### Auth and Session Hardening

- OTP-first login flow with 8-digit codes and 10-minute expiry
- `?next=` redirects are now the standard protected-route handoff
- login waits for an authoritative authenticated browser user before redirecting
- callback routing validates allowed origins and normalizes production redirects through `NEXT_PUBLIC_SITE_URL`
- missing-profile recovery now happens server-side for authenticated users whose profile row was not restored

### Change-Intelligence Runtime

- New SQL read surfaces for the Change Feed:
  - `get_change_feed_bursts`
  - `get_change_feed_burst_detail`
  - `get_change_feed_news`
- Stale queue claims are automatically requeued instead of being stranded
- Diff canonicalization reduces false positives from JSON ordering churn
- PICS history capture now retries transient failures and cools down gracefully instead of blocking latest-state upserts
